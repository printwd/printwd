FAQ

  • LAP 서버는 Linux, Apache, PHP로 구성된 서버를 뜻합니다.

    Linux는 운영 체제(OS)로, 서버가 작동하는 기반 환경을 제공합니다. 안정성, 보안성, 오픈소스 특성 덕분에 서버 환경에서 널리 사용됩니다.

    Apache는 웹 서버 소프트웨어로, 클라이언트(사용자)의 요청을 받아 웹 페이지(HTML, PHP 등)를 제공하는 역할을 합니다.

    PHP는 서버에서 실행되는 프로그래밍 언어입니다. 사용자 요청에 따라 동적인 데이터를 생성하거나, 데이터베이스와 상호작용하는 데 많이 쓰입니다.

  • SMTP 서버는 이메일을 전송하기 위해 사용되는 서버로, SMTP(Simple Mail Transfer Protocol) 프로토콜을 기반으로 동작한다. 이 서버는 이메일 발송의 핵심 역할을 하며, 사용자가 작성한 이메일을 받아 수신자의 메일 서버로 전달하거나, 다른 SMTP 서버와 통신하여 메일을 중계한다.

    SMTP 서버는 발신자의 신원을 확인하고, 도메인을 검증하며, 이메일을 올바른 경로로 라우팅하여 최종 수신자에게 도달할 수 있도록 한다. 주로 이메일 클라이언트(예: Outlook, Gmail)와 통신하며, SMTP 서버의 주소와 인증 정보는 이메일 설정에서 필수적으로 입력되어야 한다.

    이 서버는 이메일 송신 과정에서 도메인 네임 시스템(DNS)을 활용해 수신자의 메일 서버를 찾아내고, 그 서버와 연결해 이메일을 전송한다. SMTP 서버는 종종 보안 프로토콜(TLS 또는 SSL)과 결합되어 이메일 전송의 안전성을 보장하기도 한다.
